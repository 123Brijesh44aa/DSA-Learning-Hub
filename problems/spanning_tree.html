Spanning Tree & Minimum Spanning Tree (MST)

What is a Spanning Tree?

A Spanning Tree of a connected, undirected graph is a subgraph that includes all the vertices of the graph with the minimum number of edges required to connect all vertices. A spanning tree does not have cycles and cannot be disconnected.

Properties of a Spanning Tree

  • A connected graph with V vertices will have a spanning tree with exactly V - 1 edges.
  • Removing any edge from the spanning tree disconnects the graph.
  • Adding any edge to the spanning tree creates a cycle.
  • A graph can have multiple spanning trees.

What is a Minimum Spanning Tree (MST)?

A Minimum Spanning Tree (MST) is a spanning tree where the sum of the weights of the edges is minimized. In a weighted graph, different spanning trees can have different total weights. The MST is the one with the smallest total weight.

MSTs are used in network design (e.g., telephone networks, cable TV, computer networks) to minimize the cost of cabling while keeping all nodes connected.

Graph Representation in Java

To work with Spanning Trees and MSTs, we first need a way to represent a Graph. Here is a simple Edge list representation in Java.

import java.util.*; class Edge { int src, dest, weight; public Edge(int src, int dest, int weight) { this.src = src; this.dest = dest; this.weight = weight; } } class Graph { int V, E; List<Edge> edges; public Graph(int v, int e) { V = v; E = e; edges = new ArrayList<>(); } public void addEdge(int src, int dest, int weight) { edges.add(new Edge(src, dest, weight)); } }

Example

Scenario

Imagine 4 houses (0, 1, 2, 3) that need to be connected by cables. The costs to connect them are:

  • 0 - 1 : Cost 10
  • 0 - 2 : Cost 6
  • 0 - 3 : Cost 5
  • 1 - 3 : Cost 15
  • 2 - 3 : Cost 4

Goal: Connect all houses with minimum cost.

MST Selection:

  • Edge 2-3 (Cost 4)
  • Edge 0-3 (Cost 5)
  • Edge 0-1 (Cost 10)

Total Cost: 4 + 5 + 10 = 19.

This set of edges connects all houses without cycles and has the minimum possible total weight.
